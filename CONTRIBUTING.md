# Contributing to aceOS

Thank you for your interest in contributing to aceOS! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or bug fix
4. Make your changes
5. Test your changes thoroughly
6. Submit a pull request

## Development Guidelines

### Code Style

- Use consistent indentation (4 spaces)
- Follow existing naming conventions
- Add comments for complex logic
- Keep functions focused and modular

### Testing

- Test all new features in multiple browsers
- Verify that encryption/decryption works correctly
- Ensure no data leaks or security vulnerabilities
- Test on both desktop and mobile viewports

### Security

- Never commit secrets or API keys
- Ensure all data storage is encrypted
- Maintain anonymity features
- Review code for potential security issues

### UI/UX

- Maintain consistency with existing design
- Ensure smooth animations and transitions
- Test accessibility features
- Make responsive for various screen sizes

## Pull Request Process

1. Update documentation if needed
2. Add screenshots for UI changes
3. Describe your changes clearly in the PR description
4. Link any related issues
5. Wait for review and address feedback

## Areas to Contribute

- **Applications**: Complete placeholder apps (chatTrap, mailTrap, bankTrap, crapTrap)
- **Features**: Add new functionality to existing apps
- **Security**: Enhance encryption and privacy features
- **UI/UX**: Improve design and user experience
- **Documentation**: Improve guides and tutorials
- **Bug Fixes**: Resolve issues and improve stability

## Code of Conduct

- Be respectful and constructive
- Focus on the best solution for the project
- Help others learn and grow
- Respect privacy and security concerns

## Questions?

Open an issue for questions or clarification on the project.

Thank you for contributing to aceOS!
